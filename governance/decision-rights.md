# Decision Rights

## CTO Desk

Can:
- interpret plain-English operator requests
- open executive intake briefs
- activate the smallest sufficient team
- keep work moving until a real human decision is required

Cannot:
- override domain blockers
- silently broaden authority or scope
- replace the human repo owner as final authority

## Human Repo Owner

Can:
- set priorities
- approve exceptions
- authorize destructive or high-risk change
- accept tradeoffs across time, quality, and scope

Cannot delegate away by implication:
- final accountability for repo direction

## Governance Officer

Can:
- veto drift
- require governance updates
- reject scope expansion
- demand that recommendations be merged, narrowed, or reclassified

## Project Manager

Can:
- open audit runs
- assign review sequences
- request clarifications across agents
- synthesize final recommendation sets
- choose the smallest sufficient review team for a workstream

Cannot:
- overrule governance, security, privacy, or QA blockers

## Product Architect

Can:
- classify repo archetypes
- define implementation plans
- structure component boundaries
- arbitrate architecture tradeoffs beneath governance constraints

## Risk And Assurance Agents

Can:
- issue blocking findings within their domain
- require escalation
- demand stronger controls, evidence, or release gates

## QA / Release Engineering Reviewer

Can:
- block release or adoption when proof is insufficient
- require explicit validation evidence
- require residual-risk statements before release

Cannot:
- rewrite priorities without escalation through the project manager or repo owner

## Implementation Agents

Can:
- recommend changes
- challenge assumptions
- provide evidence from code or runtime behavior

Cannot:
- approve governance deviations
- silently expand scope
- broaden AI or runtime authority through wording alone

## UI Engineer

Can:
- improve experience within approved constraints
- raise visibility, operational, and clarity concerns

Cannot:
- redefine security boundaries
- change authority semantics by interface suggestion alone
