# Agency Learning Standard

## Purpose

Define how `GremTech` should keep learning across projects without drifting
into undocumented prompt behavior.

## Core Rule

`GremTech` should evolve through governed promotion, not silent mutation.

That means:
- capture useful observations
- compare them against prior work and external references
- validate what actually helps
- promote proven patterns into durable repo-owned state

## Learning Loop

1. observe
   - notice a repeated problem, strong pattern, useful reference, or better way
2. capture
   - store it in the right memory lane or work package artifact
3. compare
   - check whether the pattern already exists locally or in prior work
   - assess the source's lifecycle, support posture, and hidden cost
4. validate
   - decide whether it is reusable, safer, clearer, or faster in a defensible way
5. promote
   - move the durable part into standards, templates, docs, profiles, or long-term memory
6. reuse
   - route future projects through the promoted pattern first

## Self-Evolving Rule

`GremTech` may be self-improving in the sense that it:
- accumulates references
- remembers lessons
- updates standards
- refines templates
- improves onboarding and delivery behavior

It may not be self-modifying in the sense of:
- silently changing authority boundaries
- silently changing risk posture
- silently broadening runtime power

Those changes still need governance decisions.

## Memory Lanes

Use:
- `memory/sensory/`
  - incoming recommendations and early observations
- `memory/working/`
  - active synthesis and current learning work
- `memory/References/`
  - durable source material, exemplars, and project inspirations
- `memory/long-term/`
  - durable agency lessons and repeatable patterns
- `memory/prospective/`
  - future bets, deferred improvements, and opportunities
- `memory/consolidation/`
  - items waiting to be promoted or merged into stronger surfaces

## Promotion Targets

When a lesson becomes durable, prefer promoting it into:
- `governance/standards/`
- `templates/`
- `docs/`
- `portfolio/`

Use `memory/long-term/` when the lesson is durable but not yet mature enough
to become a standard.

## Project Start Rule

When starting a new product, major repo adoption, major infrastructure pattern,
or significant runtime/UI surface:
- look for similar local work first
- look for strong external references when useful
- record what is worth borrowing
- record what should be rejected

Use:
- [project-inspiration-standard.md](project-inspiration-standard.md)
- [selective-assimilation-standard.md](selective-assimilation-standard.md)

## Remembering Rule

Do not leave reusable learning only in:
- chat transcripts
- one-off work packages
- ad hoc notes

Capture it in a lane that future work can actually find.
