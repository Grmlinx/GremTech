# Governance

This directory defines the control layer for GremTech.

GremTech is intended to act like a governed software-development agency rather
than an unbounded swarm of prompts. That means team shape, hierarchy, decision
rights, escalation rules, role activation, repo archetypes, and recommendation
handling must all be explicit.
