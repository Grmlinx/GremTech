# gremlin-git Profile

## Archetype

Legacy tool estate / script garden

## What To Preserve

- raw utility value
- evidence of repeated operator needs
- historical scripts that can inform cleaner promoted tools
- Windows-first operator practicality
- file and folder centric workflow patterns
- willingness to use either prompt-driven scripts or richer packaged tools

## Runtime Posture

Consolidation first.
Do not expand runtime ambition before the supported tool boundary is clear.

## Default Review Lanes

- governance
- project management
- product architecture
- code review
- DevOps
- QA / release

## Standardization Focus

- catalog the estate before expanding it
- separate supported, incubating, and archived tools
- promote the strongest scripts into governed operator-tool repos
- use the estate as evidence for CLI and operator-tool standards, not as a
  template to clone directly
