# Portfolio

## Purpose

This directory records how `GremTech` sees the managed repo estate.

Use it for:
- repo classification
- repo change posture
- role activation defaults
- cross-harness posture
- runtime posture
- cross-repo adoption guidance

## Files

- [targets.json](targets.json)
  - machine-readable target-repo registry, including archetype, memory profile,
    change posture, cross-harness posture, and runtime posture
- [profiles/alfred.md](profiles/alfred.md)
- [profiles/home-lab.md](profiles/home-lab.md)
- [profiles/doofus.md](profiles/doofus.md)
- [profiles/gremlin-git.md](profiles/gremlin-git.md)
- [profiles/gremoire.md](profiles/gremoire.md)

## Rule

These files describe reusable operating truth.
They do not replace the target repos as the source of truth for their own
implementation details.
