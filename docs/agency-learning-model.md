# Agency Learning Model

## Purpose

Explain how `GremTech` should behave like a learning agency rather than a
static prompt scaffold.

## Core Model

`GremTech` should improve itself through project work.

That means each substantive engagement should be able to produce:
- better standards
- better templates
- better references
- better repo profiles
- better delivery patterns

## What Learning Means Here

Learning does not mean silent autonomous behavioral drift.

It means:
- durable capture of useful observations
- comparison with prior work and external references
- governed promotion of proven patterns
- reuse in future work

## The Memory Stack

`GremTech` now uses a hybrid agency-learning memory model:
- `memory/sensory/`
- `memory/working/`
- `memory/References/`
- `memory/long-term/`
- `memory/prospective/`
- `memory/consolidation/`

This gives the repo:
- intake
- active synthesis
- source material
- durable semantic memory
- future-looking improvement lanes
- a place to stage promotions

## Project Start Behavior

For major work, the agency should:
1. inspect the target repo
2. inspect relevant local analogues
3. inspect good external references when useful
4. capture a project inspiration scan
5. adapt the best parts into the work package

That adaptation should be selective.
The goal is to absorb the strengthening trait, not inherit the whole upstream
stack.

## Promotion Behavior

After implementation or discovery:
- move strong reusable ideas into standards or templates
- record durable lessons in long-term memory when not yet standard-ready
- keep raw inspiration in `References/`
- keep future bets in `prospective/`

## Why This Matters

Without this loop, the agency stays dependent on:
- chat memory
- operator recollection
- repeated rediscovery

With this loop, the agency becomes better over time in a way that is inspectable
and governable.
