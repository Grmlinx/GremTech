# Audit Execution Flow

## Standard Sequence

1. Intake the target repo and audit objective.
2. Governance officer defines non-negotiables.
3. Project manager defines scope and sequencing.
4. Product architect frames the system shape.
5. Risk and assurance agents review.
6. Implementation agents review.
7. Domain specialists review if needed.
8. UI engineer reviews downstream experience and visibility.
9. Project manager synthesizes findings.
10. Governance officer issues the top-level verdict.

## Output Rule

- findings first
- evidence-backed only
- recommendations to `memory\sensory\Recommendations`
- issue map updated before a new angle creates another file
