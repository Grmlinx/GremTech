# Audit Execution Flow

## Standard Sequence

1. Intake the target repo and audit objective.
2. Classify the repo archetype.
3. Product owner frames intended scope and acceptance meaning.
4. Governance officer defines non-negotiables.
5. Project manager defines scope and sequencing.
6. Product architect frames the system shape.
7. Risk and assurance agents review.
8. Implementation agents review.
9. Domain specialists review if needed.
10. UI engineer reviews downstream experience and visibility.
11. Project manager synthesizes findings.
12. Product owner decides whether the outcome still fits the intended product.
13. Governance officer issues the top-level alignment verdict.

## Output Rule

- findings first
- evidence-backed only
- recommendations to `memory\sensory\Recommendations`
- issue map updated before a new angle creates another file
- audit artifacts should record repo archetype and assessed milestone
- operator overrides should stay explicit rather than being treated as normal
  governance approval
