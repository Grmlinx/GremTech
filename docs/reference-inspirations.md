# Reference Inspirations

## Purpose

Record the main repos and systems currently shaping the `GremTech` operating
model.

This is not a vendor list or a dependency list.
It is a map of patterns worth preserving, adapting, or explicitly rejecting.

## Local Repo Inspirations

### `alfred`

Used for:
- strong `start.ai` discipline
- full operational memory model
- governed AI workflow boundaries
- executable validation, smoke, and gate discipline
- documentation-sync enforcement
- governed repo-audit control loops
- explicit release posture with support and approval categories
- machine-readable policy and assurance surfaces
- benchmark for the governed AI product platform archetype

### `home-lab`

Used for:
- governance-centered current-truth memory
- durable runbook and recovery discipline
- repo-managed persistence of operational fixes
- infrastructure verification and recovery-first operating habits

### `Gremoire`

Used for:
- human-first knowledge-vault simplicity
- clean intake versus durable reference separation

### `Doofus`

Used for:
- practical product-facing operator documentation
- lightweight tool-product orientation

### `gremlin-git`

Used for:
- evidence of repeated utility patterns that need later curation
- a reminder to separate supported tools from script-garden sprawl
- Windows-first operator-tool patterns
- interactive file and folder workflow preferences

## Rule

Take patterns, not cargo cults.

Every borrowed idea should be:
- adapted to the repo archetype
- expressed in portfolio language
- documented as a shared standard only if it improves clarity and control
