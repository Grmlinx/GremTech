# Audit Prompts Need Real Control Files

## Lesson

If a repo tells reviewers to check an issue map and backlog, those surfaces
must be real and maintained.

## Why It Matters

Otherwise recommendation discipline becomes performative: prompts reference a
tracking system that does not actually preserve overlap, ownership, or next
work.

## Reuse Rule

Future Alfred-class and audit-heavy repos should:

1. create the issue map and backlog before scaling review prompts
2. keep recommendation notes, issue consolidation, and backlog items linked
3. promote repeated recommendation themes into standards, skills, templates, or
   runtime work
