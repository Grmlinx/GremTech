# Repo-Owned Skills Are Capability Infrastructure

## Lesson

For Alfred-class work, repo-owned skill packs are part of the capability
infrastructure, not just convenience prompts.

## Why It Matters

They let repeated evaluation and build work become bounded, reusable, and
reviewable without pretending that every repeated task belongs in runtime code.

## Reuse Rule

Future Alfred-class work should:

1. identify repeated agent-facing workflows
2. decide when they belong in a skill pack instead of prose alone
3. validate them like product assets
4. keep them subordinate to governance and approval boundaries
