# Skill Catalogs Need Real Assets

## Lesson

If a repo says skill packs matter, it should have an actual skill surface and a
catalog, not only a standard that talks about them.

## Why It Matters

Otherwise the capability remains theoretical and future work falls back to ad
hoc prompts and scattered notes.

## Reuse Rule

When a skill-pack standard becomes important:

1. create the repo-owned skill surface
2. seed it with a small number of high-value packs
3. add a machine-readable catalog
4. add templates so the surface can keep growing cleanly

## Promotion Targets

- `skills/README.md`
- `governance/standards/skill-catalog.json`
- `templates/skill-pack/SKILL.md`
