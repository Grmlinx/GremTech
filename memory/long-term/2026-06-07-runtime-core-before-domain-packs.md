# Runtime Core Before Domain Packs

## Lesson

To recreate an Alfred-class system responsibly, build the governed runtime core
before expanding into heavy specialist domain packs.

## Why It Matters

If the domain packs arrive first, the repo looks sophisticated while the real
control, approval, and review boundaries are still weak.

## Reuse Rule

Future Alfred-class builds should sequence:

1. work-unit model
2. control-plane state
3. worker model
4. typed approval lane
5. validator and smoke proof
6. review API/UI
7. optional domain packs
