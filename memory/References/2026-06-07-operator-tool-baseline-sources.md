# Operator Tool Baseline Sources

## Purpose

Capture the core local source material shaping operator-tool standardization in
`GremTech`.

## Sources

### `Doofus`

Why it matters:
- shows a strong practical operator-tool product shape
- already has install, build, smoke-test, workflow, and release surfaces

What is worth borrowing:
- product-facing README discipline
- workflow-driven command organization
- practical install and release guidance

What should not be copied blindly:
- missing governance and startup surfaces

### `gremlin-git`

Why it matters:
- shows repeated operator-tool needs across many small utilities

What is worth borrowing:
- Windows-first practicality
- direct file and folder workflows
- optional interactive prompting
- progress visibility

What should not be copied blindly:
- mixed maturity
- missing supported-versus-archived boundaries
- script-garden sprawl

### `alfred`

Why it matters:
- strongest portfolio example of startup-contract rigor and memory discipline

What is worth borrowing:
- strong startup-contract thinking
- deliberate memory structure

What should not be copied blindly:
- heavier runtime and memory shape than lightweight tool repos justify
