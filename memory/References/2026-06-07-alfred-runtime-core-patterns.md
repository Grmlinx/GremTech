# Alfred Runtime Core Patterns

## Purpose

Capture the deeper runtime-core patterns observed in Alfred that matter for
future reconstruction.

## What Is Worth Borrowing

- durable work-unit record sets
- explicit control-plane state classes
- worker catalogs with execution mode and boundaries
- typed action requests plus exact approval binding
- read-only review platforms over derived records

## What Should Not Be Copied Blindly

- every Alfred path name
- every domain-heavy workflow family
- every control-plane class before the generic core actually needs it
