# Alfred Skill Surface Patterns

## Purpose

Capture the Alfred skill-surface ideas that are worth reusing in `GremTech`.

## Observed Patterns

- skills should state purpose, use conditions, inputs, outputs, allowed
  actions, disallowed actions, and data-boundary rules explicitly
- skills are stronger when they point to related standards and runbooks instead
  of trying to replace them
- skill packs are a better growth surface than multiplying agents for every new
  repeated task

## Promotion Targets

- `skills/README.md`
- `skills/registry/`
- `governance/standards/skill-catalog.json`
- `governance/standards/skill-pack-standard.md`
