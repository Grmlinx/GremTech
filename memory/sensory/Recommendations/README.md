# Recommendations

## Purpose

This folder is the intake and staging area for recommendation notes produced by
GremTech agents.

## Rule

Before creating a new recommendation file:
- check the current issue map
- check the backlog
- update or reference existing findings when they already cover the same issue
- create a new file only for a materially new review angle

## Required Metadata

Each recommendation note should include:
- From:
- Date:
- For:
- Purpose:
- Scope:
