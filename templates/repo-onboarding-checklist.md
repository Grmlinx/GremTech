# Repo Onboarding Checklist

## Classification

- repo purpose identified
- repo archetype chosen
- runtime posture chosen
- default review lanes chosen
- cross-harness need decided

## Required Surfaces

- `README.md` exists
- `start.ai` exists
- control surface exists
- documentation front door exists
- memory pattern chosen
- decisions / backlog surface exists

## Reusable Assets

- repo profile added in `portfolio/profiles/`
- target added to `portfolio/targets.json`
- relevant templates copied or adapted
- adoption steps recorded in the playbook

## Verification

- startup path is readable
- current truth is findable
- future changes can reuse the new shape
