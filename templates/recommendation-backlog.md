# Recommendation Backlog

## Purpose

Track open follow-up work implied by recommendation notes and issue-map
consolidation.

## Backlog Items

| Id | Priority | Status | Owner Role | Summary | Source | Next Durable Target |
| --- | --- | --- | --- | --- | --- | --- |
| BACKLOG-ID | P1 | open | governance-officer | short summary | issue or note | standard / template / skill / doc / runtime |

## Update Rule

When a backlog item is completed:

1. update the status
2. link the durable target that closed it
3. remove or supersede duplicate items instead of leaving parallel follow-ups
